Detailed Description

template<typename T>
class uni20::async::ReadBuffer< T >

RAII handle for reading the value from an Async container at a given epoch.

A ReadBuffer<T> represents a read-only access to the value of an Async<T> at a specific epoch. It is awaitable, and yields either a borrowed reference or an owning proxy depending on value category:

  • co_await buf yields T const&: shared read access.
  • co_await buf.transfer() yields an owning proxy that keeps the read epoch alive.
Note
transfer() is the explicit owning form for named buffers. Direct co_await on a temporary buffer still uses the owning rvalue path.
A ReadBuffer<T> can be co_awaited multiple times, but buf.transfer() transfers ownership semantics and should only be used once. After moving, further use is undefined.
