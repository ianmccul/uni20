Member Data Documentation

◆ awaiter_count_

std::atomic<int> uni20::async::BasicAsyncTaskPromise::awaiter_count_ = 0

Number of active awaiters (owners) of this coroutine.

This is exactly equal to the number of AsyncTask instances that refer to this coroutine.

Note
When the count reaches zero, the coroutine is considered unowned. Ownership must be transferred explicitly using take_ownership().

◆ cancel_on_resume_

std::atomic<bool> uni20::async::BasicAsyncTaskPromise::cancel_on_resume_ {false}

If true, then when the coroutine is next resumed (i.e.

transferred to the scheduler) it is destroyed (stack unwound and coroutine frame freed).

◆ continuation_

std::coroutine_handle<promise_type> uni20::async::BasicAsyncTaskPromise::continuation_ = nullptr

AsyncTask coroutines are nestable – we can co_await one AsyncTask inside another.

continuation_ tracks the 'parent' coroutine in this case, so that when we finish execution we return to executing the parent coroutine.

◆ eptr_

std::exception_ptr uni20::async::BasicAsyncTaskPromise::eptr_ = nullptr

◆ exception_

std::atomic<bool> uni20::async::BasicAsyncTaskPromise::exception_ {false}

EpochContext can inject an exception into a coroutine, which will be thrown when resuming from an awaiter.

This could, in principle, happen from multiple threads if we are awaiting multiple buffers so we protect access by the exception_ flag. Only the first attempt to set exception_ to true is permitted to set eptr_; subsequent exceptions are simply dropped.
