4// NOTE: Immediately-invoked coroutine lambdas must not capture variables.
5// Captures (by reference or value) are stored in the lambda frame, which is destroyed
6// after the lambda returns. If the coroutine suspends, any captured state becomes dangling.
7// Instead, pass all needed data via parameters, which are safely moved into the coroutine frame.
